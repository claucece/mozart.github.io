3 IPort Module

The IPort module provides the same functionality as the Port module, with the ability to connect to the IPort through a socket.

3.1 Introduction

The IPort module provides an Oz port that is accessible through a socket by remote Oz processes. In essence, it is the Oz flavour of what tcp socket communication usually provides. The differences are that ports are uni-directional and that they benefits from the transparent distribution of Oz. For an application, IPort provides the same functionality that Port does. An IPort can stop accepting new connections, however already existing ones are maintained.

3.2 Reference

is

{IPort.is +X ?B}

tests whether X is an IPort.

new

{IPort.new ?SocketN ?Xs ?IPort}

Opens a random free socket under which the IPort IPort is offered. SocketN is bound to the number of the socket used. Xs is bound to the stream of IPort.

newOn

{IPort.new SocketN ?Xs ?IPort}

Opens the socket SocketN (an integer) under which the IPort IPort is offered. Xs is bound to the stream of IPort.

send

{IPort.send +IPort X}

sends X to the port IPort: The stream pointed to by IPort is unified with X|_ (in a newly created thread), and the pointer advances to the stream's new tail.

SendRecv

{IPort.sendRecv +Port X Y}

sends the pair X#Y to the IPort IPort: The stream pointed to by IPort is unified with X#Y|_ (in a newly created thread), and the pointer advances to the stream's new tail.

The argument X is commonly used as message to be sent, while Y serves as reply to that message.

getFrom

{IPort.getFrom  IP SocketN ?IPort}

Binds IPort to an IPort offered on the computer whose ip adress is IP (a virtual string) on the port SocketN. An error is raised if the other computer is not reachable, or if it doesn't offer an IPort on that socket. Note that IPorts obtained using getFrom support all IPort commands (send, sendRec, is, getIP and getPN) except the stopAccepting command. Only the IPort received directly by new or newOn functions have the capability to stopAccepting. However if you give a reference to this IPort to a remote site, this site can use it to stopAccepting.

getIP

{IPort.getIP IPort ?IP}

Binds IP to a virtual string representing the ip adress of the computer that has created IPort.

getPN

{IPort.getPN IPort ?SocketN}

Binds SocketN to an integer that is the port number on which IPort is offered.

stopAccepting

{IPort.stopAccepting IPort}

Stops offering IPort to other processes. In other words, accessing to this IPort using the getFrom function will not work anymore. Note that IPorts obtained using getFrom support all IPort commands (send, sendRec, is, getIP and getPN) except the stopAccepting command. Only the IPort received directly by new or newOn functions have the capability to stopAccepting. However if you give a reference to this IPort to a remote site, this site can use it to stopAccepting.
