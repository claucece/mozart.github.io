Node features

All of the following features except for id,edges and kind/s are attributes required by daVinci, id,edges and kind/s are for the purpose of the interface.

		Node.id = Atom or Number
stands for identity of the node and is needed when specifying the edges 
		Node.'OBJECT' = Atom
will be the text shown in the node (i.e. the label) 
		Node.edges = [Edge1 ... Edge2]
 declares the outgoing edges from this node, for type of edge see below 
		Node.'INFO' = Atom
This is used to define the tooltip text that appears after a delay when the mouse pointer is over the node. 
		Node.'FONTFAMILY' = Family
Family = {lucida %default, times, helvetica, courier} This is used to visualize the text of a node in a different font. 
		Node.'FONTSTYLE' = Style
 Style = {normal, bold %default, italic, bold_italic} This attribute is used to visualize the text of a node in a different font. 
		Node.'COLOR' = Color
 can be used to define the background color of a node. Color may be any X-Window colorname (see file lib/rgb.txt in your X11 directory) or any RGB color specification in a format like '#0f331e', where 0f is the hexadecimal value for the red part of the color, 33 is the green part and 1e is the blue. The default color for nodes is white. 
		Node.'CCOLOR' = Color
 is used when the subgraph of the node is hidden, defaults to the value of 'COLOR' 
		Node.'_GO' = GraphicalObject
 GraphicalObject = {box %default, circle, ellipse, rhombus, triangle, text, invisible, icon, image, scissors} particularizes the geometrical form of the nodes,
Note that set to icon (DV Version 2.x) or image (DV-Presenter 3.0) an additional attribute (i.e. ICON or IMAGE) must be specified to declare the file name of the icon/image to be drawn into the node. scissors is usually used to display nodes with hidden subgraphs. 
		Node.'_CGO' = GraphicalObject
 similar to '_GO', used when the subgraph of the node is hidden, defaults to scissors 
		Node.'ICONFILE' = File
 This is used as a supplement for attribute '_GO' when its value is set to icon . Attribute 'ICONFILE' specifies the (relative) filename of the bitmap used to draw the node as icon. The File must be in X-Window bitmap format (.xbm). Absolute filenames are not allowed as a value for this attribute. DaVinci is looking for the File in all directories given by environment variable $DAVINCI_ICONDIR. So for example, when the bitmap of an icon is in File /home/me/images/my_image.xbm , then add /home/me/images to the search path of environement variable $DAVINCI_ICONDIR before starting daVinci, set attribute '_GO' to icon and attribute 'ICONFILE' to "my_image.xbm" for a particular node. 
		Node.'CICONFILE' = File
 similar to 'ICONFILE', used when the subgraph of the node is hidden 
		Node.'IMAGE' = File
 This is used as a supplement for attribute _GO when its value is set to image. Attribute 'IMAGE' specifies the file name of the image used to draw the node as icon. The File specified in the attribute value must be in GIF, XBM, XPM or BMP format. Absolute filenames are not allowed as a value for this attribute. DaVinci is looking for this File in all directories given by environment variable $DAVINCI_ICONDIR. So for example, when the icon is in file /home/me/images/my_image.gif (this is UNIX® notation, use C: and \ on Windows®), then add /home/me/images to the search path of environment variable $DAVINCI_ICONDIR before starting daVinci, set attribute '_GO' to image and attribute 'IMAGE' to my_image.gif for a particular node. The default value for this attribute is "". 
		Node.'CIMAGE' = File
 similar to 'IMAGE', used when the subgraph of the node is hidden 
		Node.'HIDDEN' = {true, false}
 used to hide the subgraph of a node initially, defaults to false 
		Node.'BORDER' = {none, single, double}
 defines the border of the graphical object (see attribute "_GO"), defaults to single 
		Node.m = [MenuEntry1 ... MenuEntryN]
 states the menu entries that will be shown as a popup menu at the node 
   MenuEntry = menu_entry(MenuId MenuLabel)
   MenuId    = MenuProc
   MenuProc  = proc{$ NodeId DaVinci} ... end 
   MenuLabel = Atom or Number
 MenuId identifies the action to be executed when choosing this entry, MenuLabel represents the label of the entry (i.e. what is shown at the menu bar). Note that nested menus are not implemented. The MenuId is a procedure getting two arguments, the first being the NodeId, the second denoting the DaVinci environment built from the parameters. This enviroment will provide some defaults like removing items or changing attributes. (cf.example2.oz) 
                Node.kind = Kind
 Kind = Atom refers to a self-defined layout-definition in Parameters 
		Node.kinds = [Kind1 ... KindN]
 Kind = Atom refers to a self-defined layout-definition in Parameters

(For specification of Atom and Number cf. The Oz Base Environment )
